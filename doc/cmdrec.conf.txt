* コンフィギュレーションファイル

<cmdrecコンフィギュレーションファイル> = <記録パターン>
<記録パターン> = [rec] [pattern] [{]
                 [recfile:] [recargs] <パターンマッチ演算子> <正規表現>
                 [recmatch:] [reccmd] <パターンマッチ演算子> <正規表現>
                 [}]
<パターンマッチ演算子> = ( [=~] | [!~] )
<正規表現> = スラッシュ記号で囲ったルビー的正規表現
<任意文字列> = シングルクオーテーションマークで囲ったルビー的文字列
<再生パターン名> = シングルクオーテーションマークで囲ったルビー的文字列

注意

スペースは何個あってもよく、改行やタブを含んでもよい。
# より後はコメント

例

-------- cmdrec.conf --------
rec pattern {
	recfile:
		recargs =~ /^.+\.c$/
	recmatch:
		reccmd =~ /\/(clang.*|gcc.*|g++.*)$/
}
-----------------------------

* 記録時パラメータ

reccwd ...... 記録時のカレントディレクトリ
　　　　　記録時のレコードが無かった場合は playcwd と同じ内容となる。
reccmd ...... 記録時のコマンド名(第ゼロ引数)
　　　　　記録時のレコードが無かった場合は空になる。
recfile ..... 記録パターンの recfile: にマッチした引数
　　　　　一つのみでなければエラー。
　　　　　相対パスになっている場合は、reccwd を付加してフル＆トゥルーパスに変換。
　　　　　ファイルは実際に存在しなければならない。
　　　　　記録時のレコードが無かった場合は空になる。
recargs ..... 記録時の引数配列
　　　　　記録パターンの recfile: で使用した場合は、記録時のコマンド名(第ゼロ引数)は削
　　　　　除される。
　　　　　その他で使用した場合は、記録時のコマンド名(第ゼロ引数)と、記録パターンの
　　　　　recfile: にマッチした引数は削除される。
　　　　　要素区切りを "\n" に置き換え。最後にも "\n" が有る。
　　　　　記録時のレコードが無かった場合は空になる。

* 記録アルゴリズム(cmdrec)

記録時パラメータを用意する。
記録パターンで検索し、recfile が丁度一つだけマッチし、そのファイルが存在し、reccmd が
マッチした場合に限って、コマンドデータベースに記録時パラメータを記録する。
recfile が存在しなかった場合はその旨表示してエラーコードを返す。
実際に記録時パラメータを execve する。
